# 常见假设模式

本文档提供常见 bug 类型对应的假设模式和埋点位置建议。

## 假设生成原则

1. **宁多勿少** - 生成 3-5 个假设，覆盖不同方向
2. **奥卡姆剃刀** - 简单假设优先
3. **最近变更** - 优先考虑最近修改的代码
4. **复现条件** - 与复现步骤相关的代码路径
5. **历史经验** - 类似问题的历史记录

## 前端常见问题

### UI 闪烁/抖动

**症状**：页面元素短暂出现后消失或改变

**假设模式**：
| 假设 ID | 假设描述 | 埋点位置 |
|--------|---------|---------|
| H1 | 组件多次渲染 | 组件函数入口，记录渲染次数 |
| H2 | 状态更新导致重渲染 | setState/useState 调用处 |
| H3 | 条件渲染逻辑错误 | 渲染条件判断处 |
| H4 | CSS 动画/过渡问题 | 样式变化触发点 |

### 数据加载失败

**症状**：页面显示加载中或错误状态

**假设模式**：
| 假设 ID | 假设描述 | 埋点位置 |
|--------|---------|---------|
| H1 | API 请求失败 | fetch/axios 调用前后 |
| H2 | 请求超时 | 请求开始/结束时间 |
| H3 | 响应数据格式错误 | 响应数据解析处 |
| H4 | 错误处理逻辑问题 | catch/error handler |
| H5 | 认证/权限问题 | 请求头/token 检查 |

### 状态不同步

**症状**：UI 显示与实际数据不一致

**假设模式**：
| 假设 ID | 假设描述 | 埋点位置 |
|--------|---------|---------|
| H1 | 状态更新时机错误 | 状态更新调用处 |
| H2 | 异步操作顺序问题 | 异步操作开始/结束 |
| H3 | 组件未正确订阅状态 | 状态订阅/监听处 |
| H4 | 缓存数据过期 | 缓存读取/写入处 |

### 事件处理无响应

**症状**：点击/输入等操作无反应

**假设模式**：
| 假设 ID | 假设描述 | 埋点位置 |
|--------|---------|---------|
| H1 | 事件处理函数未绑定 | 组件挂载时的事件绑定 |
| H2 | 事件被阻止传播 | stopPropagation 调用 |
| H3 | 条件阻止了处理 | 事件处理函数入口 |
| H4 | 异步操作阻塞 | 处理函数内的异步调用 |

### 内存泄漏/性能问题

**症状**：页面变慢、卡顿

**假设模式**：
| 假设 ID | 假设描述 | 埋点位置 |
|--------|---------|---------|
| H1 | 未清理的定时器 | setInterval/setTimeout |
| H2 | 未取消的订阅 | 订阅创建/销毁处 |
| H3 | 大数据量渲染 | 列表渲染处 |
| H4 | 重复请求 | API 调用处 |

## 假设生成示例

### 示例 1：用户登录失败

**用户描述**：点击登录按钮后一直转圈，没有反应

**假设生成**：

```markdown
## 问题假设

### H1: [最可能] API 请求超时
- 位置: `src/api/auth.ts:login()`
- 原因: 登录 API 响应时间过长
- 验证: 记录请求开始/结束时间

### H2: 请求未发出
- 位置: `src/components/LoginForm.tsx:handleSubmit()`
- 原因: 表单验证阻止了请求
- 验证: 记录 handleSubmit 是否执行

### H3: 响应处理错误
- 位置: `src/api/auth.ts:login()` 的 catch 块
- 原因: 错误被静默捕获
- 验证: 记录是否进入 catch 块

### H4: 状态更新问题
- 位置: `src/store/authStore.ts:setUser()`
- 原因: 状态更新后未触发 UI 更新
- 验证: 记录状态变化

### H5: 按钮事件未绑定
- 位置: `src/components/LoginForm.tsx` 按钮 onClick
- 原因: 事件处理器未正确绑定
- 验证: 记录按钮点击事件
```

### 示例 2：数据保存失败

**用户描述**：编辑表单后点击保存，提示成功但数据没有更新

**假设生成**：

```markdown
## 问题假设

### H1: [最可能] 请求数据格式错误
- 位置: `src/api/user.ts:updateUser()`
- 原因: 发送的数据格式与后端期望不匹配
- 验证: 记录请求 body 内容

### H2: 后端返回成功但未实际保存
- 位置: 后端 `UserController.update()`
- 原因: 后端逻辑有条件跳过保存
- 验证: 记录后端处理流程

### H3: 前端未正确处理响应
- 位置: `src/components/UserForm.tsx:handleSave()`
- 原因: 成功响应后未刷新数据
- 验证: 记录响应后的处理流程

### H4: 缓存导致显示旧数据
- 位置: `src/hooks/useUser.ts`
- 原因: 本地缓存未更新
- 验证: 记录缓存更新时机

### H5: 乐观更新回滚
- 位置: `src/store/userStore.ts`
- 原因: 乐观更新后因某原因回滚
- 验证: 记录状态变化历史
```

## 埋点数量建议

| 假设数量 | 建议埋点数 | 说明 |
|---------|----------|-----|
| 1-2 | 2-3 | 聚焦单一问题 |
| 3-4 | 4-6 | 典型调试场景 |
| 5+ | 6-8 | 复杂问题，注意不超过 10 |

## 假设排序原则

1. **从用户行为到系统响应** - 按执行顺序排列
2. **概率高的优先** - 常见问题放前面
3. **容易验证的优先** - 简单埋点能验证的先做
4. **最近变更优先** - 刚修改的代码更可能有问题
